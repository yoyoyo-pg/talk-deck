# 教訓・再発防止ルール

> **使い方**: セッション開始時に必ず一読する（session-init.sh が自動注入）。ユーザーから指摘・修正を受けたら即座に追記する。

## エントリ形式

```
### [ルール名（動詞で始まる命令形）]
**ルール**: 具体的に何をする/しない
**なぜ**: 過去に起きた問題・インシデント
**適用条件**: どのときに発動するか
```

- **「何をしたか」は書かない**。「次回どうするか」を命令形で書く
- コードや git log で分かることは書かない
- ユーザーから指摘されたら、指摘の内容をそのまま記録するのではなく、**再発を防ぐルール**として抽象化して書く
